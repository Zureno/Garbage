Finding 8: Using Components with Known Vulnerabilities
         (Fingerprint2.js Library)
================================================================================

SEVERITY: MEDIUM-HIGH

Dependency: Application

Status: Open

Description:
The application includes and actively executes Fingerprint2 v1.5.1, an outdated
library known for device fingerprinting and persistent user tracking. This 
component is loaded from a publicly accessible file and collects comprehensive
device information to generate unique identifiers that enable permanent user
identification across sessions.

The library represents a vulnerable dependency that should be removed from
production code due to its known privacy implications and lack of user consent
for data collection.

Affected Systems & Locations:

File: /gettaxcalcbac/javascript/fingerprint2.js
URL: https://qa.gmsgetb.kpmglinkgo.com/gettaxcalcbac/javascript/fingerprint2.js
Library: Fingerprint2 v1.5.1
Release Date: 2015 (10+ years old)
File Size: ~15KB unminified
Access: PUBLIC (no authentication required)
Status: ACTIVELY LOADED AND EXECUTED IN PRODUCTION

Reproduction Steps:

Step 1: Access the Component File
  URL: https://qa.gmsgetb.kpmglinkgo.com/gettaxcalcbac/javascript/fingerprint2.js
  Method: Open in browser or download
  Result: Complete outdated Fingerprint2 library is publicly accessible

Step 2: Verify Active Execution
  Navigate to any application page
  Open Browser DevTools → Console
  Execute:
    new Fingerprint2().get(function(result, components) {
        console.log("Fingerprint ID:", result);
        console.log("Components collected:", components.length);
    });
  Result: Library actively executes and collects device data

Step 3: Monitor Data Collection
  DevTools → Network tab
  Observe AJAX POST to: /WebService/SetFingerprinting
  Request Body: { fingerprinting: "unique_device_id_hash" }
  Result: Collected data transmitted to server

Step 4: Verify Persistence
  Clear all cookies and cache
  Close and reopen browser
  Execute Step 2 code again
  Observation: SAME fingerprint ID generated
  Result: Tracking persists beyond session/cookies

Step 5: Confirm Known Vulnerabilities
  Library: Fingerprint2 v1.5.1 (2015)
  Status: Unmaintained/deprecated
  Known Issues:
    • Privacy concerns with device fingerprinting
    • No user consent mechanism
    • Permanent device identification
    • Cannot be disabled by users
    • Violates user privacy principles

Technical Details:

COMPONENT ANALYSIS:

Library Name: Fingerprint2
Version: v1.5.1
Release Year: 2015
Maintenance Status: DEPRECATED/UNMAINTAINED
Repository: Deprecated on GitHub
Current Version: v2.x exists, but fingerprinting is discouraged

KNOWN VULNERABILITIES IN THIS COMPONENT:

1. Device Fingerprinting Implementation
   Purpose: Identify devices uniquely
   Impact: Enables permanent user tracking
   Scope: Affects all users

2. Permanent Identifier Generation
   Data Points: 50+ device characteristics
   Combinations: Results in unique per-device ID
   Persistence: Survives cookie deletion
   Scope: Permanent until hardware changes

3. Silent Data Collection
   Disclosure: No user notification
   Consent: No opt-in mechanism
   Control: No user ability to disable
   Awareness: Invisible to users

4. Biometric-like Data Collection
   Canvas Fingerprinting: GPU rendering patterns
   WebGL Fingerprinting: Graphics capabilities
   Hardware Info: CPU cores, memory
   System Info: OS, timezone, plugins
   Exposure: Sensitive device information

THIRD-PARTY COMPONENT RISK:

Dependency Type: External JavaScript library
Dependency Management: Not apparent in codebase
Version Pinning: Hard-coded to v1.5.1
Update Path: No update mechanism visible
Security Monitoring: Likely no vulnerability scanning

Step 4: Exploit - Extract and Store Fingerprint
new Fingerprint2().get(function(result) {
    // Extracted device fingerprint
    var fingerprint = result;
    console.log("Device Fingerprint:", fingerprint);
    
    // In production, this is sent to server for persistent tracking
    $.ajax({
        url: "/WebService/SetFingerprinting",
        type: "POST",
        data: { fingerprinting: fingerprint }
    });
});

SECURITY IMPACT:

User Tracking:
  • Users can be tracked indefinitely via device fingerprint
  • Tracking persists across sessions and browsers
  • Impossible for users to prevent or disable
  • Enables behavioral profiling over time

Device Reconnaissance:
  • Complete hardware profile collected (GPU, CPU, RAM)
  • Software configuration exposed (installed fonts, plugins)
  • System information harvested (OS, timezone, language)
  • Browser capabilities profiled (extensions, storage)

Session Linking:
  • Multiple sessions linked to same device
  • User behavior correlated across time
  • Activity timeline constructed per device
  • Unauthenticated sessions linked to authenticated

Account Security Risk:
  • Fingerprint can be used to identify compromised accounts
  • Device profiling enables fraud detection bypass
  • Multi-account fraud detection weakened
  • Account takeover scenarios enabled

Data Breach Risk:
  • If fingerprint database breached, permanent identifiers exposed
  • Users cannot change/reset fingerprints
  • Enables user profiling and targeting
  • Biometric-like data disclosed

SEVERITY JUSTIFICATION:

Risk Rating: MEDIUM-HIGH

Factors Increasing Severity:
  • Component is 10+ years old (v1.5.1 from 2015)
  • Publicly accessible file enables easy discovery
  • Active in production code (executing on all pages)
  • Collects sensitive device biometric data
  • No user consent or ability to disable
  • Enables permanent user identification
  • Data breach would expose permanent identifiers

Factors Potentially Reducing Severity:
  • Privacy issue rather than code injection vulnerability
  • Not directly exploitable for system compromise
  • Requires user to have session on application
  • Library itself is not "vulnerable" in technical sense

Overall Assessment: MEDIUM-HIGH
  - Components with known privacy risks = Medium
  - Sensitive data collection without consent = High
  - Persistence and tracking capability = High
  - User impact potential = High

SUMMARY

The application contains an outdated, deprecated JavaScript library
(Fingerprint2 v1.5.1) that:

1. Collects comprehensive device information
2. Generates permanent, unique device identifiers
3. Enables permanent user tracking across sessions
4. Is publicly accessible and easily discoverable
5. Represents a known vulnerable component
6. Should be removed from production immediately

This is a clear example of "Using Components with Known Vulnerabilities"
where a deprecated library with known privacy risks is actively executing
in production code.

Remediation required: REMOVE fingerprinting library immediately and replace
with proper server-side session management.
